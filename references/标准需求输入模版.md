**标准需求输入模版（AI 热点日报）**

把下面这段直接复制作为开场提示，并替换方括号中的内容。

---

我需要你在本地创建一个“AI 热点日报”系统，包含抓取、生成、排版、分发的完整流程。请按以下要求执行并给我交付：

**1. 数据来源与排序**
1. 视角优先级：产品 > 市场 > 技术 > 工具 > 研究
2. 权重排序：Karpathy Blog > 官方公司 > 其他 > 论文
3. 默认数据源：RSS/官方源（不要依赖 X API 为主）
4. 过滤关键词：剔除含 [不关注关键词列表] 的内容

**2. 输出结构（必须严格）**
每条热点结构如下（全部中文）：
```
1. 【重要】【产品视角】<a href="原文链接">标题</a>
a) 内容概要：
- …
- …
b) AI PM关注点：
- …
- …
c) 如何设计：
- …
- …
d) 市场反馈：
- …
- …
e) 标签：
- …
f) 评注：
- …
- …
```

**3. 内容要求**
1. 内容概要 / AI PM关注点 / 如何设计：每块约 100 字
2. 评注：顶级大模型产品经理视角，约 100 字
3. 关键短语高亮格式：`<span style="background-color:#FFF2CC;">…</span>`
4. 标签只保留与大模型直接相关的关键词

**4. 交付形式**
1. HTML 输出，直接可在手机端阅读
2. 生成并打开本地预览：`/tmp/wechat_article.html`
3. 如果网络可用：尝试发送到 PushPlus；不可用时保留 HTML 并说明原因

**5. 发布与自动化**
1. 微信公众号草稿发布（仅草稿，不强制发布）
2. PushPlus 微信推送（token 由我提供）
3. DNS 或网络失败时：先写清原因再暂停

**6. 附加**
1. 把所有规则写入技能文档，方便后续每天自动化调用
2. 任何密钥不要写入仓库

---

**可选变量（按需替换）**
1. [不关注关键词列表] 示例：濒危物种, 基因, 生物多样性
2. [每日条数] 示例：10
3. [时间窗口] 示例：24 小时 / 48 小时
4. [输出语言] 示例：中文
